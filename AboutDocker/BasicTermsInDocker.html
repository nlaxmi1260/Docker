Basic Terms in Docker

  1. Web Servers : A web server is a software application (or hardware device) that stores, processes, and delivers web pages to clients (usually web browsers) over the Internet.
    A web server’s primary role is to serve web pages for a website . A web page can be rendered from a single HTML file, or a complex assortment of resources fitted together. If you want to host your web application on the internet, in many cases you will need a web server.
    The most popular open source web servers are currently Apache and Nginx .
  2. Containerisation: The process of containing applications, along with its code and libraries in one environment or container,running on more than one platform to make the DevOps delivary pipeline simplified and portable is known as containerization.
      Key Concepts In Containerisation:
    1. Containers: Containers are lightweight, standalone and executable software packages that include everything needed to run a piece of software, including the code, runtime , system tools ,liabraries and settings.
    2. Docker: The most popular platform for containerization is Docker. Docker provides tools to create ,deploy and manage containers. It uses a DockerFile to define the steps to create a container image, which is a snapshot of the container.
        Key concepts & Components of Docker
      1. Docker Images :
        1. Definition: A Docker image is a lightweight, standalone, and executable software package that includes everything needed to run a piece of software: the code, runtime, libraries, environment variables, and configuration files.
        2. Characteristics: Images are read-only; once built, they cannot be modified directly. They are layered, meaning each image can inherit from another image, reducing redundancy.
      2. Docker Containers
        1. Definition: A Docker container is a runnable instance of a Docker image. It is an isolated environment where the application runs, sharing the host OS kernel but with its own file system, processes, and network interfaces.
        2. Characteristics: Containers can be started, stopped, moved, and deleted independently. They can be ephemeral (temporary) or persistent, depending on how they are configured.
      3. Dockerfile :
        1. Dockerfile Definition: A Dockerfile is a text file that contains a series of instructions for building a Docker image. It specifies the base image to use, the application files to include, and the commands to run when creating the image.
        2. Common Instructions: FROM: Specifies the base image.
          RUN: Executes commands in the image during the build process.
          COPY or ADD: Copies files from the host into the image.
          CMD or ENTRYPOINT: Specifies the command to run when a container is started from the image.
        3. Example: A Dockerfile that installs a web server and copies application files into the image.
      4. Docker Hub:
        1. Definition: Docker Hub is a cloud-based registry service for sharing and storing Docker images. It allows users to upload their images and download images created by others.
        2. Features: Public and private repositories for images. Official images maintained by Docker and the community. Versioning and tagging of images for easier management.
        3. Example: Pulling the official nginx image from Docker Hub using docker pull nginx.
      5. Volume:
        1. Definition: A Docker volume is a persistent storage mechanism used to store data generated and used by Docker containers. Volumes are managed by Docker and exist outside the container’s file system.
        2. Characteristics: Data in volumes can persist even after containers are stopped or deleted. Volumes can be shared between multiple containers.
        3. Example: Using a volume to store a database’s data files so that they remain intact between container restarts.
      6. Network:
        1. Definition: Docker networks provide communication between containers and the host system. Docker allows you to create isolated networks for containers to communicate.
        2. Types of Networks:
          1. Bridge: The default network mode for Docker containers; allows containers to communicate on the same host.
          2. Host: Uses the host's networking stack directly.
          3. Overlay: Allows containers on different Docker hosts to communicate, useful for multi-host setups (like Docker Swarm).
          4. macvlan
          5. None
          6. Ipvlan
        3. Example: Creating a bridge network to allow web and database containers to communicate.
      7. Docker Compose:
        1. Definition: Docker Compose is a tool for defining and managing multi-container Docker applications using a docker-compose.yml file. It allows you to specify the services, networks, and volumes for your application.
        2. Features: Simplifies the orchestration of multiple containers. Supports defining services, scaling, and managing the lifecycle of the application.
        3. Example: Using Docker Compose to run a web application alongside a database in separate containers defined in a single YAML file.
      8. Docker Swarm:
        1. Definition: Docker Swarm is Docker's native clustering and orchestration tool, allowing you to manage a cluster of Docker engines as a single virtual Docker engine.
        2. Characteristics: Provides high availability and load balancing. Allows scaling of services up or down easily.
        3. Example: Deploying a multi-container application across several Docker hosts for improved reliability and performance.
      9. Docker Registries:
        1. Definition: A Docker registry is a storage and distribution system for Docker images. Docker Hub is a public registry, while you can also set up private registries.
        2. Functionality: Allows pushing and pulling of images. Manages image versions and access control.
        3. Example: Using a private Docker registry to store proprietary images for internal use.
      10. Orchestration
        1. Definition: Orchestration refers to the automated management of containerized applications, including deployment, scaling, and networking. Tools like Kubernetes and Docker Swarm handle orchestration.
        2. Purpose: Ensures that applications are deployed in the desired state. Manages resource allocation and load balancing.
        3. Example: Using Kubernetes to orchestrate a microservices architecture where different services are deployed in separate containers.
