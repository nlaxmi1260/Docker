Docker Commands

    To become a Docker expert, you must understand key commands for
  1. Basic Docker Commands: These commands cover the core functionality for working with containers, images, and volumes .
    • docker --version: Display the installed Docker version.
    • docker info: Display system-wide information about the Docker installation, including the number of containers, images, and the storage driver in use.
    • docker help: Show help for Docker commands.
  2. Container Lifecycle Management :
      Managing the lifecycle of containers — running, stopping, and removing is crucial for containerized applications :
    • docker build -t < Image_name > . : To create image from the dockerFile. Here . refers to dockerfile in the current directory
    • docker run:Start a new container from an image. You can pass different options to control the container behavior:
      • docker run -it < Image_name > : Start an interactive container
      • docker run -d < Image_name >: Run in detached mode(background)
      • docker run --name < ContainerName > < ImageName >: Start a container with a specific container name.
      • docker run --network < network-name > < image-name > : Run a container attached to a specific network.
      • docker run -d -p 8080:80 <image-name >: 8080 is host port and 80 is the container port
      • docker run -volume
      • docker run --subnet
      • docker run
    • docker ps: List running containers.
      • docker ps -a: List all containers (including stopped)
    • docker stop : Stop a running container. docker stop < container-id >
    • docker start : Start a stopped container. docker start < container-id >
    • docker rm : Remove a container. Containers must be stopped before removing them.
      • docker rm < container-id >
      • docker rm $(docker ps -a -q) : Remove all stopped containers
    • docker kill: Forcefully stop a running container.
      • docker kill < container-id >
  3. Working with Images : Docker images are the core of containerization. Managing and manipulating images effectively is a key skill:
    • docker images: List all available images on your local machine.
      docker images
    • docker pull: Download an image from Docker Hub or a custom registry.
      docker pull < image-name >
    • docker build: Build an image from a Dockerfile.
      1. docker build -t .
      2. docker build -t dockerfilename
    • docker rmi: Remove one or more images.
      docker rmi < image-id/imageName >
    • docker tag: Tag an image for easy reference.
      docker tag <image-id > < repository >:< tag >
    • docker push: Push an image to a remote registry like Docker Hub.
      docker push < repository >:< tag >
  4. Container Interaction: Interacting with running containers is essential for debugging and working with containerized applications
    • docker exec: Run commands inside a running container.
      docker exec -it < container-id > /bin/bash
    • docker attach: Attach your terminal to a running container’s output.
      docker attach < contaner-Id >
    • docker logs: View the logs of a running or stopped container.
      docker logs < container-id >
    • docker inspect: Get detailed information about a container or image in JSON format.
      docker inspect < contaner_id >
  5. Networking : Understanding how Docker networking works is key to deploying and managing containers that need to communicate
    • docker network ls: List all Docker networks.
    • docker network create < network-name > : Create a new Docker network.
    • docker network inspect < network-name > : Display detailed information about a specific network.
    • docker network connect < network-name > < container-id > :Connect a container to a network.
    • docker run --network < network-name > < image-name> : Run a container attached to a specific network.
  6. Volumes & Storage:Containers are ephemeral(having a very short life cycle) by design, but volumes are used to persist data between container restarts or across multiple containers:
    • docker volume ls: List all Docker volumes
    • docker volume create < volume-name >: Create a new volume.
    • docker run -v <volume-name >:/path/in/container <image-name > : Mount a volume when running a container
    • docker volume rm < volume-name > : remove a volume
    • docker volume inspect < volume-name > : Get detailed information about a volume.
  7. Container Orchestration (Docker Compose):Docker Compose simplifies multi-container applications, letting you define services, networks, and volumes in a single file.
    • docker compose up -d :Start all services defined in the docker-compose.yml file in detached mode.
    • docker compose down :Stop and remove all containers and networks defined in the compose file.
    • docker compose logs: View logs from all services.
    • docker-compose exec < service-name > <command > : Run a command inside a running service container.
    • docker compose build Build or rebuild services.
  8. Advanced Docker Commands: To truly become a Docker expert, you need to understand more advanced commands and concepts for troubleshooting, performance optimization, and cleanup:
    • docker prune Clean up unused Docker resources (containers, networks, volumes, and images).
      docker system prune
      docker volume prune
    • docker stats:Monitor the real-time performance of running containers.
    • docker history < image-id >Show the history of an image, including layers and commands used to create the image
    • docker cp < source-path >< container-id >:<destination-path > Copy files between the host and a container.
    • docker commit < container-id > < new-image-name > Create a new image from a container's changes.
  9. Security Best Practices :Securing Docker containers and images is an important aspect of containerization
    • docker scan < image-name >: Scan Docker images for known vulnerabilities (with Docker Hub integration)
    • docker run --cap-drop=ALL --name < container-name > <image-name >: Drop unnecessary Linux capabilities to reduce the container’s attack surface.
    • docker run --read-only < image-name > : Run a container with a read-only file system.
